# Правила ведения второго мозга

Прочитай один раз — и пиши по этим правилам. Цель: чтобы и ИИ-агент, и обычный человек открыли любой файл и сразу поняли, *что это, насколько свежо и актуально ли сейчас*.

---

## 1. Шапка файла (frontmatter)

Каждый содержательный `.md`-файл начинается с такого блока:

```yaml
---
created: 2026-06-27      # дата создания, не меняется
updated: 2026-06-27      # дата последнего изменения, обновляй при правках
status: active           # active | idea | archived
tags: [стратегия, mvp]   # необязательно, для поиска по темам
---
```

### Статусы
- **active** — актуально, на повестке, влияет на решения *сейчас*.
- **idea** — сырая мысль/гипотеза, ещё не в работе.
- **archived** — остыло. Оставляем для истории, но это уже не повестка.

Дата + статус — это и есть ответ на «что важно, а что просто архив».

> **Источник истины об актуальности — поле `status`, и только оно.** Список «На повестке» в [index.md](index.md) — это витрина (удобные ссылки на главное), он может отставать от реальности. Если данные расходятся — верь статусу в самом файле, а не витрине.

---

## 2. Архив: у каждой папки свой

Архив **локальный** — в каждом разделе есть файл `_archive.md`. Остывшее не уносим в общую дальнюю папку, а оставляем рядом с темой. Так в любой папке всё просто: **в корне — актуальное, в `_archive.md` — остывшее.**

Две степени:

1. **Лёгкий архив** — файл остался на месте, но `status: archived`. Контекст рядом, история под рукой.
2. **Глубокий архив** — переносим в `_archive.md` своего раздела. Короткую заметку — целиком под датой; большую (отдельный файл) — оставляем файлом со статусом `archived` и ставим на неё ссылку из `_archive.md`.

Правило простое: **сначала меняем статус, переносим в `_archive.md` только когда корень раздела начинает захламляться.**

---

## 3. Имена файлов

- Латиница/кириллица — на твой выбор, но единообразно. По умолчанию: **кириллица в нижнем регистре, слова через дефис**: `позиционирование-zametna.md`.
- Заметки в `journal/` начинай с даты: `2026-06-27-мысли-о-онбординге.md`. Так дневник сам сортируется по времени.

---

## 4. Ссылки = граф

Связывай файлы обычными markdown-ссылками относительно текущего файла:

```markdown
Подробнее в [гипотезе про онбординг](../hypotheses/онбординг-за-3-экрана.md).
```

Не дублируй текст — лучше сошлись. Дерево папок задаёт порядок, ссылки задают смысл.

---

## 5. Где что писать

- **goals/** — 🎯 главное. Цели по трём горизонтам: текущая, краткосрочные, долгосрочные. Всё остальное работает на них.
- **strategy/** — *как* достигаем целей: видение, позиционирование, крупные решения и их обоснование. (Цели — это «что и зачем», стратегия — «как».)
- **initiatives.md** — 🚀 все большие задачи в **одном файле** (исключение из правила «папка на раздел» — ради фокуса). Инициатива — это *действие* («что делаем»), в отличие от гипотезы (*вопрос*); связывай их ссылками. Одна инициатива = блок `###`, который двигаешь между секциями: 🔥 В фокусе (по приоритету) → 📋 Бэклог → ✅ Сделано → 🗄 Архив.
- **hypotheses/** — формат «предположение → как проверяем → результат → вывод». Одна гипотеза = один файл.
- **analytics/** — факты и цифры: рынок, пользователи, метрики, конкуренты. То, на что можно опереться.
- **journal/** — поток мысли по датам. Сюда можно писать сыро; ценное потом «поднимается» в strategy/hypotheses/analytics.
- **`_archive.md`** в каждой папке — остывшее этого раздела (см. п.2).

Сомневаешься, куда положить, — клади в `journal/` с датой и поставь ссылку из нужного раздела. Лучше записать, чем потерять.

---

## 6. Для ИИ-агента: рабочий цикл

1. Начни с [`index.md`](index.md), затем этот файл.
2. Чтобы понять «что сейчас» — смотри блок «На повестке» в `index.md` и файлы со `status: active`.
3. Создавая файл — добавь frontmatter с сегодняшней датой.
4. Меняя файл — обнови `updated` и, если надо, `status`.
5. Сделал что-то значимое для повестки — отрази строкой в «На повестке» в `index.md`.
6. Нашёл связь — поставь ссылку.
